## 1 引言

### 1.1 项目背景
传统人工填报、线下汇总模式存在数据上报不及时、统计口径不统一等痛点，影响人社部门研判与监管实效。为规范数据采集、审核、汇总、上报全流程，实现省、市、企业三级数据互联互通，保障数据真实可追溯，特建设本系统，实现与国家失业监测系统无缝对接，为全省就业失业工作提供信息化支撑。

### 1.2 编写目的
明确系统功能、业务流程、数据规则、接口规范等核心内容，为系统全生命周期工作提供统一依据，确保需求清晰可落地，避免开发与需求脱节，交付符合业务及监管要求的信息化系统。

### 1.3 项目范围
- **用户范围**：企业用户（备案、填报、查询）、市级管理员（辖区数据审核、通知发布）、省级管理员（全省统筹管理），权限严格隔离。
- **业务范围**：涵盖企业备案、数据填报、市级审核、省级汇总上报、数据分析可视化，实现业务闭环。
- **支撑范围**：包含通知管理、用户权限管理、系统监控、数据接口、编码管理等支撑功能。

### 1.4 术语定义
- **调查期**：默认按月统计，省级管理员可调整，同步更新填报、审核规则。
- **建档期**：企业备案审核通过时间，作为数据对比基准，不可随意修改。
- **上报时限**：省级设定各环节截止时间，逾期禁止操作，特殊情况可补报。
- **级联**：下拉框关联逻辑，父选项变化时子选项自动筛选，如地区、失业原因选择。

## 2 总体描述

### 2.1 用户角色
- **企业用户**：完善企业备案并上报，填报调查期数据，查询本企业历史数据，修改账号密码，查看各级通知。
- **市级用户**：审核辖区企业数据，退回错误数据并说明原因，发布辖区通知，修改自身密码，仅可操作本市数据。
- **省级用户**：审核全省企业备案，管理三级用户及角色，终审数据并上报部委，开展多维度数据分析，配置系统参数、监控系统运行。

### 2.2 业务总流程
1. 省级管理员创建企业账号并下发，企业用户登录后完善备案信息并提交省级审核。
2. 备案审核通过后，企业在调查期内填报数据并提交市级审核；审核不通过则退回修改后重报。
3. 市级审核通过后，数据同步至省级，省级终审通过后汇总上报国家部委；终审不通过则退回市级重审。
4. 省级管理员对汇总数据进行多维度分析，生成报表及可视化图表，支撑政策制定。

## 3 功能需求

### 3.1 功能1：企业上报数据审核
#### 3.1.1 功能概述
为本模块核心功能，市局管理员可对辖区内企业上报的月度就业失业数据进行查看、核验、审核通过、退回修改操作，完成数据的市级审核流转，是数据上报省局的必经环节。

### 3.2 企业备案信息
#### 3.2.1 基本描述
企业用户首次登录系统后，需补充完善企业基础信息，并上报至省局进行备案。备案通过后，企业方可进行每月就业失业数据填报。若备案被退回，企业可修改后重新上报。

#### 3.2.2 数据说明
企业备案信息数据项及规则如下：

| 数据项 | 必填 | 说明 |
|--------|------|------|
| 所属地区 | 是 | 显示企业所属地市、市县、区域，采用三级级联下拉选择，不可手动修改 |
| 组织机构代码 | 是 | 仅可输入字母、数字，长度不超过9位；全省范围内唯一，系统自动校验是否重复 |
| 企业名称 | 是 | 支持中文、英文，长度不超过100字符 |
| 企业性质 | 是 | 两级下拉选择，第一级为"国有、民营、外资、其他"，第二级根据第一级动态加载 |
| 所属行业 | 是 | 两级下拉选择，参照国家标准行业分类 |
| 主要经营业务 | 是 | 文本输入，不超过200字符 |
| 联系人 | 是 | 支持中文、英文，长度不超过50字符 |
| 联系地址 | 是 | 两级下拉选择（地市、区县），详细地址文本输入（不超过100字符） |
| 邮政编码 | 是 | 6位数字，格式校验 |
| 联系电话 | 是 | 格式符合"区号-电话号码"或11位手机号码，系统校验格式 |
| 传真 | 是 | 格式符合"区号-电话号码"，系统校验格式 |
| EMAIL | 否 | 符合邮箱格式 xxx@xxx.xxx |

#### 3.2.3 相关功能
- **保存**：暂存填写信息，状态为"未上报"，可随时修改。
- **上报**：将当前信息提交至省局备案，状态变更为"待审核"。上报后不可修改，直至省局审核。
- **修改**：备案被退回后，企业可修改信息并重新上报，系统保留修改记录。

### 3.3 数据填报
#### 3.3.1 基本描述
企业用户每月在省局规定的调查期时间范围内，填报当期就业失业数据。填报完成后可暂存，确认无误后上报至市局。上报后若市局未审核，企业可撤回修改；若市局已审核，则不可修改。

#### 3.3.2 数据说明
| 数据项 | 必填 | 说明与校验规则 |
|--------|------|----------------|
| 建档期就业人数 | 是 | 正整数，首次建档时填写的监测点就业人数，系统自动带出不可修改 |
| 调查期就业人数 | 是 | 正整数，当期实际就业人数，系统自动计算增减变化 |
| 调查期新增就业人数 | 是 | 正整数，当期新增就业人数 |
| 调查期减少就业人数 | 是 | 正整数，当期减少就业人数 |
| 调查期末失业人数 | 是 | 正整数，当期期末失业人数 |
| 失业原因分类 | 是 | 多选下拉，包含"经济性裁员"、"合同到期"、"个人原因"等选项 |
| 备注说明 | 否 | 文本输入，不超过500字符，用于说明特殊情况 |

## 6 系统管理
### 6.1 用户管理
- 省级管理员可创建、修改、禁用企业、市级、省级用户账号
- 支持批量导入企业用户
- 支持密码重置功能

### 6.2 权限管理
- 基于角色的访问控制（RBAC）
- 三级权限体系：省级、市级、企业级
- 功能权限与数据权限分离

### 6.3 系统监控
- 用户登录日志
- 数据操作日志
- 系统性能监控
- 异常告警机制

## 7 非功能需求
### 7.1 性能要求
- 系统响应时间：普通操作≤3秒，复杂查询≤10秒
- 并发用户数：支持1000人同时在线
- 数据存储：支持5年历史数据存储

### 7.2 安全性要求
- 数据传输加密（HTTPS）
- 用户密码加密存储
- 防止SQL注入、XSS攻击
- 操作日志审计

### 7.3 可用性要求
- 系统可用性≥99.5%
- 支持7×24小时运行
- 故障恢复时间≤4小时

### 7.4 兼容性要求
- 浏览器兼容：Chrome 80+、Firefox 75+、Edge 80+
- 移动端适配：支持主流手机浏览器
- 操作系统：Windows 10+、macOS 10.15+、主流Linux发行版

## 9 异常处理规则
### 9.1 数据异常
- 数据格式错误：系统提示具体错误位置和原因
- 数据逻辑错误：系统进行逻辑校验并提示
- 数据重复提交：系统检测并提示

### 9.2 系统异常
- 网络异常：自动重试机制，失败后友好提示
- 服务器异常：错误页面引导，记录详细日志
- 数据库异常：事务回滚，数据一致性保证

### 9.3 业务异常
- 超时未操作：自动保存草稿，会话超时提示
- 权限不足：友好提示并引导至有权限页面
- 业务流程中断：支持从断点继续
