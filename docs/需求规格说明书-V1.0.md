1 引言
1.1 项目背景
传统人工填报、线下汇总模式存在数据上报不及时、统计口径不统一、监管滞后等痛点，影响人社部门数据研判与监管实效。为规范数据采集、审核、汇总、上报全流程，实现省、市、企业三级数据互联互通，保障数据真实可追溯、全程留痕、可监管，特建设本系统，实现与国家失业监测系统无缝对接，为全省就业失业监测、分析、决策提供信息化支撑。
1.2 编写目的
明确系统功能、业务流程、数据规则、接口规范、编码体系、安全与性能要求，为开发、测试、验收、运维提供统一、可落地、可追溯的依据，确保系统交付符合业务与监管要求。
1.3 项目范围
用户范围：企业用户、市级管理员、省级管理员，三级权限严格隔离。
业务范围：企业备案、月度 / 1–3 月半月度数据填报、多级审核、省级汇总、国家上报、数据分析可视化。
支撑范围：通知管理、用户 / 权限管理、系统监控、数据接口、地区编码、数据字典、异常处理。
1.4 术语定义
调查期：数据统计周期；常规为月度，1–3 月增加半月度。
半月度周期：1–3 月每月分为上半月（1–15 日）、下半月（16 日至月末）。
建档期：企业备案审核通过时间，为数据对比基准，不可随意修改。
上报时限：省级配置截止时间，逾期禁止操作，特殊情况可申请补报。
级联：下拉框联动，父选项变化子选项自动筛选。
周期类型：月度 / 半月度（上半月 / 下半月）。
2 总体描述
2.1 用户角色
企业用户
备案上报、数据填报、撤回修改、历史查询、密码修改、通知查看。
市级用户
辖区企业数据审核、退回并注明原因、通知发布、数据查看、权限隔离。
省级用户
企业备案审核、账号管理、数据终审、汇总上报国家、数据分析、系统配置、监控。
2.2 业务总流程
省级创建企业账号 → 企业备案 → 省级审核备案。
备案通过 → 企业按周期填报：
4–12 月：按月上报
1–3 月：按月 + 半月度双周期上报
企业上报 → 市级审核 → 省级终审 → 汇总上报国家。
省级可视化分析、报表导出、数据监管。
3 功能需求
3.1 企业备案信息
3.1.1 基本描述
企业首次登录必须完成备案，备案通过方可填报数据。退回可修改重报。
3.1.2 备案数据项
表格
数据项	必填	规则
所属地区	是	省–市–县三级级联，不可手动输入
组织机构代码	是	9 位字母 / 数字，全省唯一
企业名称	是	100 字符以内
企业性质	是	国有 / 民营 / 外资 / 其他 → 二级下拉
所属行业	是	国标行业分类二级下拉
主要经营业务	是	200 字符以内
联系人	是	50 字符以内
联系地址	是	地市 + 区县 + 详细地址
邮政编码	是	6 位数字
联系电话	是	固话 / 11 位手机格式校验
传真	是	固话格式
电子邮件	否	邮箱格式
3.1.3 备案状态
未上报 → 待审核 → 审核通过 / 审核退回（可修改重报）
3.2 数据填报（月度 + 半月度）
3.2.1 周期规则
4–12 月：每月 1 期
1–3 月：每月 2 期（上半月 / 下半月），独立填报、独立审核、独立上报
3.2.2 填报数据项
表格
数据项	必填	规则
建档期就业人数	是	自动带出，不可修改
调查期就业人数	是	正整数
调查期新增就业人数	是	正整数
调查期减少就业人数	是	正整数
调查期末失业人数	是	正整数
失业原因分类	是	多选：经济性裁员 / 合同到期 / 个人原因 / 其他
周期类型	是	系统自动标识：月度 / 上半月 / 下半月
备注说明	否	500 字符以内
3.2.3 填报操作
暂存、上报、撤回（未审核可撤回）、查看历史、导出。
3.3 数据审核（市级 + 省级）
3.3.1 市级审核
查看辖区企业列表、数据详情
操作：审核通过 / 退回修改（必须填写原因）
审核后同步至省，企业不可再修改
3.3.2 省级终审
全省数据汇总、查看、校验
终审通过 → 可上报国家
终审退回 → 退回市级重审
3.4 半月度上报管理（需求变更）
3.4.1 自动周期
1、2、3 月系统自动开启半月度模式，无需手动开关。
3.4.2 截止时间
上半月：每月 15 日 24:00
下半月：每月 最后 1 个工作日 24:00
节假日自动顺延
3.4.3 提醒规则
截止前 3 个工作日 自动推送系统通知。
3.4.4 数据隔离
月度、半月度数据分开存储、分开统计、分开上报。
3.5 通知管理
省级发布全局通知
市级发布辖区通知
企业可查看、标记已读
支持上报提醒、审核结果提醒、截止提醒
3.6 数据分析与可视化
按地区、行业、周期类型统计
图表：柱状图、折线图、排名表
支持导出 Excel/PDF
支持月度 / 半月度切换统计
4 数据接口
4.1 接口用途
与国家失业监测系统对接，实现数据上报、回执接收、状态同步。
4.2 接口类型
全量上报
增量上报
状态查询
回执解析
4.3 接口协议
传输：HTTPS
认证：API Key + 签名
报文：JSON
编码：UTF-8
4.4 上报字段（核心）
企业统一编码、地区编码、行业编码
周期类型（月度 / 上半月 / 下半月）
调查期、建档期人数、新增 / 减少 / 失业人数
失业原因编码、上报时间、审核状态
4.5 异常处理
超时重试、失败日志、重发机制、回执校验。
5 云南省地区编码应用规则
5.1 编码结构
6 位行政区域编码：
前 2 位：省级（53 云南省）
中间 2 位：市级
后 2 位：区 / 县级
5.2 级联规则
省级 → 市级 → 区 / 县级，下拉联动，不可跨区域选择。
5.3 使用场景
企业备案所属地区
数据权限隔离（市局只能看本市）
数据汇总统计
国家接口上报编码映射
5.4 维护规则
省级管理员统一维护，支持新增、修改、禁用、同步。
6 系统管理
6.1 用户管理
省级创建 / 禁用 / 重置密码
批量导入企业账号
账号状态：正常 / 停用 / 锁定
6.2 权限管理（RBAC）
三级权限：省级 → 市级 → 企业
数据权限：市级仅看本市
功能权限：按钮级 / 菜单级控制
6.3 周期配置管理
配置 1–3 月半月度开关
配置上报截止时间
配置提醒提前天数
6.4 系统监控
登录日志
数据操作日志（增 / 删 / 改 / 审 / 报）
性能监控、在线人数、接口调用统计
异常告警
7 非功能需求
7.1 性能
普通操作 ≤ 3 秒
复杂查询 ≤ 10 秒
并发 ≥ 1000 用户
支持 5 年数据存储
7.2 安全
HTTPS 加密
密码加密存储
防 SQL 注入、XSS
操作日志审计
数据备份（本地 + 异地）
7.3 可用性
可用性 ≥ 99.5%
7×24 小时运行
故障恢复 ≤ 4 小时
7.4 兼容性
Chrome 80+、Firefox 75+、Edge 80+
Windows10+、macOS 10.15+
支持手机浏览器浏览
8 数据字典与编码规则
8.1 核心数据字典
用户信息表、企业信息表、备案表、数据填报主表、明细表、审核日志表、上报日志表、通知表、地区编码表、行业编码表、失业原因编码表。
8.2 编码规则
周期类型编码
1：月度
2：上半月
3：下半月
失业原因编码
01：经济性裁员
02：合同到期
03：个人原因
99：其他
数据状态编码
00：暂存
01：已上报待审核
02：市级通过
03：市级退回
04：省级通过
05：省级退回
06：已上报国家
地区编码
国标 6 位行政区域码。
9 异常处理规则
9.1 数据异常
格式错误：提示字段位置
逻辑错误：数值校验不通过
重复上报：禁止提交并提示
9.2 系统异常
网络异常：自动重试 + 友好提示
服务异常：统一错误页 + 日志记录
数据库异常：事务回滚，保证一致
9.3 业务异常
超时未操作：自动保存草稿
权限不足：禁止操作并提示
流程中断：支持断点继续
